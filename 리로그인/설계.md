# 인곽민원창구 — 리로스쿨 로그인 설계

> 인곽위키(`번외_인곽위키`)의 리로스쿨 인증 방식을 분석하여, 민원창구용으로 간단하게 재설계한 문서입니다.
> 리로스쿨은 공식 OAuth/API를 제공하지 않으므로, **서버가 브라우저를 흉내 내어 직접 로그인하고 사용자 정보 HTML을 파싱**하는 방식을 사용합니다.

---

## 1. 목표

- 민원창구에 글을 쓰는 사람이 **실제 우리 학교 구성원(재학생/졸업생/교사)인지** 확인한다.
- 리로스쿨 ID/PW는 **인증 순간에만 사용하고 절대 저장하지 않는다.**
- 인증 결과(이름·학번·기수·직책)만 우리 DB/세션에 저장한다.

---

## 2. 전체 흐름

```
[사용자]
   │  ① 리로스쿨 ID/PW 입력
   ▼
[민원창구 서버]  POST /riro/login
   │  ② 리로스쿨에 대신 로그인 (스크래핑)
   ▼
[리로스쿨 서버 iscience.riroschool.kr]
   │  ③ token 발급 → 프로필 HTML 반환
   ▼
[민원창구 서버]
   │  ④ HTML 파싱 → 이름/학번/기수/직책 추출
   │  ⑤ 세션에 인증정보 저장 (ID/PW는 버림)
   ▼
[사용자]  → 민원 작성 가능
```

---

## 3. 핵심 모듈 — `riro_auth.py`

리로스쿨 인증의 심장. 인곽위키 `riroschoolauth.py`를 단순화한 형태.

### 3.1 인증 절차 (3단계 HTTP 요청)

| 단계 | 요청 | 목적 |
|------|------|------|
| ① 로그아웃 | `POST /user.php?action=user_logout` | 기존 세션 초기화 |
| ② 로그인 | `POST /ajax.php` (app=user, mode=login, id, pw) | `token` 발급 (JSON 응답) |
| ③ 프로필 | `POST /user.php` (쿠키 `cookie_token=token`) | 사용자 정보 HTML 획득 |

**로그인 응답 코드**
- `code == "000"` → 성공, `token` 사용
- `code == "902"` → 아이디/비밀번호 오류
- 그 외 → 실패

### 3.2 HTML 파싱으로 추출하는 값

| 항목 | 위치 | 비고 |
|------|------|------|
| 이름 | `.input_disabled[0]` | |
| 학번(raw) | `.input_disabled[1]` | 3자리 이상이면 `raw[0] + raw[2:]`로 변환 |
| 직책 | `span.m_level3` 또는 `span.m_level1` | "재학생"/"졸업생"/"교사" 등 |
| 기수 | ID 앞 2자리로 계산 | `int("20"+id[:2]) - 1994 + 1` |

> 계정 유형이 **통합아이디(integrated)** 인 경우 `.elem_fix` 셀렉터로 별도 파싱.

### 3.3 안정성

- 네트워크/파싱 실패 시 **최대 5회 재시도**(2초 간격).
- 모든 입력값이 채워지고 `기수 > 0`일 때만 성공으로 인정.

### 3.4 반환 형식

```python
# 성공
{"status": "success", "name": "홍길동", "student_number": "1203",
 "generation": 32, "role": "재학생"}

# 실패
{"status": "error", "message": "아이디 또는 비밀번호가 틀렸습니다."}
```

## 4. 라우트 — `POST /riro/login`

```
1. POST로 riro_id, riro_pw 수신
2. riro_auth.check_login(id, pw) 호출  ← 블로킹이므로 executor/스레드로 실행
3. 성공:
     - 직책에 '교' 포함 & '학'·'졸' 없음 → 교사
     - 그 외 → 학생/졸업생 (기수 0이면 '졸업생'으로 보정)
     - 세션에 {name, student_number, generation, role, verified=True} 저장
     - ID/PW는 메모리에서 폐기
4. 실패: 에러 메시지 반환
```

---

## 5. 데이터 — 세션에만 저장

| 키 | 예시 | 설명 |
|----|------|------|
| `riro_verified` | `True` | 인증 완료 플래그 |
| `riro_name` | `"홍길동"` | 실명 |
| `riro_student_number` | `"1203"` | 학번 |
| `riro_generation` | `32` | 기수 |
| `riro_role` | `"재학생"` | 직책 |

> **DB에는 ID/PW를 절대 저장하지 않는다.** 민원 작성 시 위 세션 값을 작성자 신원으로 첨부한다.

## 6. 최소 구현 스켈레톤

```python
# riro_auth.py
import requests, time
from bs4 import BeautifulSoup

BASE = "https://iscience.riroschool.kr"
HEADERS = {
    "Content-Type": "application/x-www-form-urlencoded",
    "User-Agent": "Mozilla/5.0 ... Chrome",
}

def check_login(user_id: str, user_pw: str, retries: int = 5) -> dict:
    s = requests.Session()
    for _ in range(retries):
        try:
            # ① 로그아웃
            try:
                s.post(f"{BASE}/user.php?action=user_logout", timeout=10)
            except requests.RequestException:
                pass

            # ② 로그인 → token
            r = s.post(f"{BASE}/ajax.php", headers=HEADERS, timeout=15, data={
                "app": "user", "mode": "login", "userType": "1",
                "id": user_id, "pw": user_pw, "deeplink": "", "redirect_link": "",
            })
            j = r.json()
            code = str(j.get("code"))
            if code == "902":
                return {"status": "error", "message": "아이디 또는 비밀번호가 틀렸습니다."}
            if code != "000":
                return {"status": "error", "message": f"로그인 실패 code={code}"}
            token = j.get("token")
            if not token:
                return {"status": "error", "message": "토큰 없음"}

            # ③ 프로필 HTML
            r2 = s.post(f"{BASE}/user.php", headers=HEADERS, data={"pw": user_pw},
                        cookies={"cookie_token": token}, allow_redirects=False, timeout=15)
            soup = BeautifulSoup(r2.text, "html.parser")

            # ④ 파싱
            el = soup.select_one("span.m_level3") or soup.select_one("span.m_level1")
            inputs = soup.select(".input_disabled")
            if not el or len(inputs) < 2:
                raise RuntimeError("파싱 실패, 재시도")

            role = el.get_text(strip=True)
            name = inputs[0].get_text(strip=True)
            raw  = inputs[1].get_text(strip=True)
            student_number = raw[0] + raw[2:] if len(raw) >= 3 else raw
            generation = int("20" + user_id[:2]) - 1994 + 1 if user_id[:2].isdigit() else 0

            if all([name, student_number, role]) and generation > 0:
                return {"status": "success", "name": name,
                        "student_number": student_number,
                        "generation": generation, "role": role}
            raise RuntimeError("데이터 누락, 재시도")
        except (requests.RequestException, RuntimeError, ValueError):
            time.sleep(2)
    return {"status": "error", "message": "인증 서버와 통신 중 오류가 발생했습니다."}
```

```python
# route_login.py (Flask 예시)
import asyncio, flask
from riro_auth import check_login

async def riro_login():
    if flask.request.method != "POST":
        return flask.render_template("riro_login.html")

    rid = flask.request.form.get("riro_id", "")
    rpw = flask.request.form.get("riro_pw", "")
    loop = asyncio.get_running_loop()
    result = await loop.run_in_executor(None, check_login, rid, rpw)  # 블로킹 분리

    if result.get("status") != "success":
        return f"<script>alert('{result['message']}');history.go(-1);</script>"

    role = result["role"]
    is_teacher = ("교" in role) and ("학" not in role) and ("졸" not in role)

    flask.session.update({
        "riro_verified": True,
        "riro_name": result["name"],
        "riro_student_number": result["student_number"],
        "riro_generation": result["generation"],
        "riro_role": "교사" if is_teacher else role,
    })
    return flask.redirect("/minwon/write")  # 민원 작성 페이지로
```

## 7. 주의사항

- **비공식 스크래핑**: 리로스쿨이 HTML 구조나 `ajax.php` 스펙을 바꾸면 파싱이 깨진다 → 셀렉터를 상수로 모아 관리하고, 실패 시 알림.
- **블로킹 처리**: `requests`는 동기 호출이므로 비동기 서버에서는 반드시 executor/스레드로 분리.
- **속도 제한**: 짧은 시간에 반복 인증 시 리로스쿨 측 차단 가능 → 사용자당 재시도 제한 권장.
- **보안**: ID/PW는 로그·DB·예외 메시지에 절대 남기지 않는다. HTTPS 필수.
- **개인정보**: 세션에는 인증에 필요한 최소 정보만 저장.
